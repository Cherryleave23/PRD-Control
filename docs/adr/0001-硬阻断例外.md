# 硬阻断例外：违反不变式触发自动返工

本生态默认"软提醒不阻断"——所有偏离提示用户后继续执行，保留开发者控制权。但对"违反不变式"这一类偏离，破例采用硬阻断：agent 自动返工，不提示操作者。

理由：不变式是系统在任何情况下都要满足的约束（如"订单总额始终非负"），违反它意味着实现与设计边界完全相反，系统进入未定义状态，可能引发数据损坏或级联故障。让用户"知情后继续"在此场景下风险过高——用户可能低估不变式违反的下游成本，或误以为是可控偏离而放行。硬阻断把这类高风险偏离的处置权交给 agent，确保系统约束不被轻易破坏。

这是"不阻断"原则的唯一例外，边界严格限定为"违反不变式"一类——越界、缺斤少两、触碰范围外仍为软提醒。

## Considered Options

- 全部软提醒不阻断（原方案）——风险在于不变式违反被轻视
- 全部硬阻断——风险在于夺走开发者控制权，违反本生态核心理念
- **硬阻断仅限违反不变式（采纳）**——平衡：高风险偏离自动处置，其他偏离保留用户控制权

## Consequences

- agent 需能可靠识别"违反不变式"与其他三类偏离的边界——依赖 SPEC.md 不变式段落写得精确可判。
- 用户失去对不变式违反的即时决策权——通过 SPEC.md 不变式段落的精确表达间接补偿（写 SPEC 时就把约束定清楚）。
- 硬阻断返工的代码仍记入 `docs/deviations.md`，保留可追溯性。
