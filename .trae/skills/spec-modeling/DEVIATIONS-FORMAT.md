# DEVIATIONS-FORMAT.md 格式

`docs/deviations.md` 是项目偏离记录的单一真相源。由 `spec-modeling` 在实现期写入，收尾时审查。SPEC.md 不含偏离记录段落。

## 关键约束

`docs/deviations.md` 是偏离记录的**唯一存放点**——SPEC.md、CONTEXT.md、ADR 均不得含偏离记录段落（不变式 I3）。单一真相源，避免维护漂移。

## 两类偏离

| 类别 | 触发场景 | 处理方式 |
|---|---|---|
| 实现偏离 | 实现期代码与 SPEC 条目不一致 | 按根因×后果矩阵处理 |
| 规范偏离 | 实现期发现 SPEC 条目本身有误 | 状态回退 `implementing → draft`，重访冲突条款 |

## 实现偏离的四类型

- **越界**：实现了 SPEC 未要求的东西
- **缺斤少两**：漏实现 SPEC 要求的东西
- **违反不变式**：破坏了系统在任何情况下都要满足的约束 → 硬阻断自动返工
- **触碰范围外清单**：实现了 SPEC 显式标注"不做"的功能

## 根因×后果诊断

### 根因（Root Cause）

- **实现意外**（incidental）—— 代码层面疏忽，本意未想偏离
- **返工缺陷**（rework）—— 偏离源于既有实现缺陷，需返工修复
- **模型自添加设计**（model-added）—— 模型自行扩展了 SPEC 未要求的设计
- **其他**（other）—— 需用户说明

### 后果（Consequence）

- **功能正常**（functional）—— 偏离后该功能仍能正常运作
- **功能异常**（broken）—— 偏离后该功能无法正常运作

### 处理建议矩阵

| 根因 \ 后果 | 功能正常 | 功能异常 |
|---|---|---|
| 实现意外 | 软提醒，建议回滚越界部分 | 强提醒，建议修复 |
| 返工缺陷 | 强提醒，建议返工 | 强提醒，建议返工 |
| 模型自添加设计 | 软提醒，询问：纳入 SPEC（回阶段①）或移除 | 强提醒，建议移除并重做 |
| 其他 | 软提醒，听用户决定 | 强提醒，听用户决定 |

**硬阻断例外**：违反不变式（任意根因/任意后果）直接自动返工，不走矩阵，不提示操作者。

## 五字段记录格式

每条偏离记录含五个字段，编号 `D1`、`D2`...：

```markdown
### D1

- **类型**：违反不变式 | 越界 | 缺斤少两 | 触碰范围外 | 规范偏离
- **根因**：实现意外 | 返工缺陷 | 模型自添加设计 | 其他（说明）
- **后果**：功能正常 | 功能异常
- **处理决定**：自动返工 | 软提醒+用户决定（说明） | 状态回退（规范偏离）
- **日期**：YYYY-MM-DD
```

## 完整文件模板

```markdown
# 偏离记录

本文件是项目偏离记录的单一真相源。由 spec-modeling 在实现期写入，阶段 ④ 收尾时审查。

## 实现偏离

### D1

- **类型**：越界
- **根因**：模型自添加设计
- **后果**：功能正常
- **处理决定**：软提醒，用户选择纳入 SPEC，回阶段 ① 更新边界
- **日期**：2026-07-19

### D2

- **类型**：违反不变式
- **根因**：实现意外
- **后果**：功能异常
- **处理决定**：自动返工（硬阻断，未提示操作者）
- **日期**：2026-07-19

## 规范偏离

### D3

- **类型**：规范偏离
- **根因**：SPEC 条目 I2 与现实冲突（不变式不可满足）
- **后果**：N/A（非实现偏离）
- **处理决定**：状态回退 implementing → draft，重访 I2，已写代码保留待评估
- **日期**：2026-07-19
```

## 编号约定

- 偏离记录编号 `D1`、`D2`...，按出现顺序，一旦分配不复用。
- 删除条目时留空号或标注"已归档"，避免引用错位。
- 收尾时（阶段 ④）审查所有偏离，决定哪些需更新 SPEC、哪些归档。
